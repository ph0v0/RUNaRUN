# Git 推送规则

- 默认推送到 `main` 分支
- 命令: `git push origin master:main`

# 版本号规则

- 当前版本: `v1.0.1`
- 位置: `index.html` 中的 `#version` 元素
- 每次提交时递增版本号（第三位数字 +1）
- 格式: `v1.0.0` → `v1.0.1` → `v1.0.2` ...

# 功能开发规则

- 每次完成新功能后，需要反向编写该功能的文档说明

# Git 提交规则

- 每次完成一个任务后，直接发布到 Git 仓库

# 错误记录规则

- 执行过程中遇到的错误、无法执行的操作，记录到专门的文档中
- 避免在新会话中重复踩相同的坑
